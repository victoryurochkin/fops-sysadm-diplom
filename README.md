# 1. Введение

В современных условиях веб-проекты должны обеспечивать не только корректную работу приложения, но и:

- устойчивость к отказам,
- защиту от сетевых атак,
- централизованный контроль состояния,
- прозрачность логирования.

Целью данной работы является построение полноценной отказоустойчивой инфраструктуры для веб-сайта **victoryur.tech**, включающей:

- внешний Web Application Firewall (WAF);
- балансировку нагрузки между несколькими серверами;
- централизованный мониторинг;
- систему агрегирования и визуализации логов.

Вся инфраструктура развернута на базе гипервизора Proxmox VE, а маршрутизация и фильтрация трафика реализованы с использованием межсетевого экрана OPNSense. Настройка всех компонентов выполнена вручную с применением открытого программного обеспечения. От использования средств автоматизации -Ansible было сознательно решено отказаться, поскольку целью работы являлось детальное изучение архитектуры каждого сервиса, принципов их взаимодействия и механизмов конфигурации на низком уровне. Такой подход позволил глубже проработать сетевые, серверные и прикладные аспекты инфраструктуры, а также продемонстрировать понимание процессов развертывания и интеграции без абстракции, создаваемой инструментами оркестрации.

# 2. Цели и задачи

## Цель работы

Спроектировать и реализовать отказоустойчивую, безопасную и наблюдаемую среду для размещения веб-сайта.

## Реализация задачи будет осуществлятся с помощью таких технологий как:

- Внешний WAF (BunkerWeb).
- Внутренний балансировщик Nginx.
- Два идентичных бэкенд-сервера.
- Система мониторинга Zabbix.
- Стек Elasticsearch + Kibana.
- Filebeat для отправки логов.
- Резервное копирование виртуальных машин на базе гипервизора proxmox.

# 4. Настройка внешнего WAF (BunkerWeb)

WAF развернут на Ubuntu 22.04 с использованием Docker.

## Установка Docker

```bash
apt update && apt upgrade -y
curl -fsSL https://get.docker.com | sh
systemctl enable docker --now
```

## Запуск контейнера

```bash
docker run -d \
  --name bunkerweb-aio \
  -v bw-storage:/data \
  -p 80:8080/tcp \
  -p 443:8443/tcp \
  -p 443:8443/udp \
  bunkerity/bunkerweb-all-in-one:1.6.7
```

Настроено проксирование:

- `victoryur.tech` → `http://192.168.1.90:80`

# 5. Настройка балансировщика Nginx

На сервере `192.168.1.90` установлен Nginx.

## Конфигурация upstream

```nginx
upstream victoryur_backend {
    least_conn;
    server 192.168.1.88:80 max_fails=2 fail_timeout=5s;
    server 192.168.1.89:80 max_fails=2 fail_timeout=5s;
    keepalive 32;
}
```

Алгоритм балансировки: **least_conn**.
